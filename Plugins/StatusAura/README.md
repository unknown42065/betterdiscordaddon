# StatusAura

A BetterDiscord plugin that reads your **Spotify** or **YT Music** activity and dynamically shifts Discord's entire accent color palette to match the album art. Plugin and live theme combined in one file — no separate `.theme.css` needed.

> **Requires [Unknown654Lib](../Unknown654Lib/) to be installed and enabled first.**

---

## Features

- **Live palette extraction** — samples the album art image on a canvas to find the most vibrant colors
- **Full brand scale override** — generates all `--brand-experiment-*` CSS variables (100–900) from the palette hue, so every Discord UI element that uses the accent color updates
- **Smooth transitions** — color shifts animate across the UI at a configurable speed
- **Spotify support** — hooks into Discord's internal `SpotifyStore` + `SPOTIFY_PLAYER_STATE` dispatcher for reliable, low-latency updates (requires Spotify to be connected to your Discord account)
- **YT Music support** — reads YT Music rich presence activity from apps like PreMiD or standalone RPC clients
- **Configurable intensity** — Subtle / Balanced / Vivid controls how aggressively colors are applied
- **Per-element toggles** — individually enable or disable effects on buttons, scrollbars, mentions, badges, and background tint
- **Graceful fallback** — if the album art can't be loaded (CORS, network, etc.) it silently does nothing rather than crashing

## Installation

1. Install and enable **[Unknown654Lib](../Unknown654Lib/)** first.
2. Download `StatusAura.plugin.js`.
3. Place it in your BetterDiscord plugins folder:
   - **Windows:** `%AppData%\BetterDiscord\plugins\`
   - **macOS:** `~/Library/Application Support/BetterDiscord/plugins/`
   - **Linux:** `~/.config/BetterDiscord/plugins/`
4. Enable **StatusAura** in Settings → Plugins.

## YT Music Setup

Discord doesn't natively expose YT Music as a presence. You need a third-party RPC app to bridge the two. Recommended options:

| App | Platform | Notes |
|-----|----------|-------|
| [PreMiD](https://premid.app/) | Browser extension | Works on Chrome/Firefox; needs the YouTube Music presence script enabled |
| [YouTube Music Discord RPC](https://github.com/XerxesNoble/youtube-music-discord-rpc) | Desktop | Standalone app, no browser needed |
| [discord-music-presence](https://github.com/ungive/discord-music-presence) | Desktop | Supports multiple players including YT Music |

Once any of these are running and showing your status in Discord, StatusAura picks it up automatically.

## Settings

Open the settings panel via Settings → Plugins → StatusAura → ⚙

| Setting | Options | Description |
|---------|---------|-------------|
| Transition speed | Fast / Medium / Slow | How quickly colors animate when the track changes |
| Intensity | Subtle / Balanced / Vivid | How strongly the palette is applied across the UI |
| Affect buttons | Toggle | Color primary action buttons |
| Affect scrollbars | Toggle | Color scrollbar thumbs |
| Affect mentions & highlights | Toggle | Tint mention rows and inline code borders |
| Affect notification badges | Toggle | Color unread badges, role tags, speaking rings |
| Subtle background tint | Toggle | Very faint palette tint on sidebar (experimental) |

The settings panel also shows a live status indicator — green dot when a source is active, with the source name shown.

## How it works

### Color extraction
The album art URL is drawn onto a 48×48 off-screen canvas. Each pixel is scored by a *vibrancy* metric (weighted saturation + distance from extreme brightness). The top-scoring pixel becomes the **primary** color. A secondary color with a hue at least 35° away is also selected for future use. Tonal variants (light, dark, muted) are derived from the primary hue using HSL math.

### CSS application
From the primary hue, a complete brand scale is generated by holding the hue constant and stepping through a predefined lightness table — matching the shape of Discord's own `--brand-experiment-*` scale. This means every Discord component that consumes any brand variable (buttons, focus rings, checkboxes, active states, etc.) automatically picks up the new color.

### Source priority
1. **Spotify** — `SpotifyStore` (fastest, via Discord's native Spotify integration)
2. **Spotify fallback** — presence activity (if SpotifyStore is unavailable)
3. **YT Music** — presence activity from a third-party RPC bridge

When music stops, the theme is removed and Discord returns to its default colors.
